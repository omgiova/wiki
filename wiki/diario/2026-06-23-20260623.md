## Configuração Dashboard Basic Auth — 13:43

O usuário precisou configurar autenticação básica (username/password) no dashboard Hermes após uma atualização que passou a exigir auth. O dashboard em `2.24.121.135:9119` estava bloqueando acesso porque `basic_auth` estava vazio no config.yaml.

**Configuração final:**
- username: `omgiova`
- password_hash: scrypt gerado via `plugins.dashboard_auth.basic.hash_password('15071995')`
- plugin `dashboard_auth/basic` adicionado em `plugins.enabled`

**Pitfalls encontrados e resolvidos:**
1. Dashboard exige `password_hash` (não `password` em plaintext) — erro "Refusing to bind dashboard to 0.0.0.0 — no auth providers registered"
2. Plugin `dashboard_auth/basic` **não carrega automaticamente** — precisa estar em `plugins.enabled` (opt-in)
3. Não consigo reiniciar `hermes-dashboard.service` de dentro do gateway — erro "cannot restart or stop the gateway from inside the gateway process". Reinício deve ser via SSH direto no VPS
4. `hermes config set` serializa arrays como string YAML com escapes (`"[\\n  spotify\\n  dashboard_auth/basic\\n]"`). Precisei corrigir manualmente via Python para formato YAML correto
