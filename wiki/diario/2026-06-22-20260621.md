## setMessageReaction — técnica documentada — 00:39

Endpoint: `POST https://api.telegram.org/bot<TOKEN>/setMessageReaction`

Parâmetros: `chat_id`, `message_id`, `reaction` (array, máx 1 para bots), `is_big`.

Emojis suportados: lista de ~70 emojis do Telegram. 🐙 (polvo) **NÃO** é válido — retorna `REACTION_INVALID`. Usar 🔥 como alternativa testada.

Para obter message_id: extrair do link `t.me/c/{chat_id_sem_-100}/{thread_id}/{message_id}`. Alternativa: `HERMES_SESSION_MESSAGE_ID` (pendente validação).

## Extração de message_id via link — 00:40

O link `https://t.me/c/3870518428/1/537` revela a arquitetura de forum do Telegram:
- `3870518428` → chat_id = `-1003870518428` (prefixo `-100` para supergroups)
- `1` → thread/topic_id (tópico "General" ou sem nome)
- `537` → **message_id** sequencial global do grupo

Método confiável: pedir ao usuário "Copiar link" da mensagem → extrair message_id do último segmento da URL. Funciona mesmo quando `getUpdates` retorna vazio (gateway consome updates).
