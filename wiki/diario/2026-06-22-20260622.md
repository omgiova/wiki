## Nova técnica: teste sequencial para descoberta de message_id — 01:37

Como solução para o problema de message IDs desatualizados, desenvolvemos uma técnica de teste sequencial: quando a variável de sessão falha, testamos IDs próximos a ela (em uma faixa configurável, por exemplo ±5) usando uma reação conhecida por ser válida (como 👍). Essa abordagem:
1. Parte da variável de sessão como linha de base
2. Testa IDs em ambas as direções alrededor dela
3. Para no primeiro ID válido encontrado
4. Funciona mesmo quando a variável de sessão está desatualizada devido a timing issues
Essa técnica foi encapsulada no skill `telegram-bot-api` como Método 5 e também implementada como um script auxiliar `scripts/find-message-id.sh` para uso repetido, incorporando a preferência do usuário por auto-descoberta em vez de solicitação de informações externas.

## Correção de comportamento: estabelecimento de protocolo de reações seguras — 04:52

O usuário corrigiu comportamentos específicos do agente durante o teste de reações:
- Reagir às próprias mensagens (inaceitável - o agente deve apenas reagir às mensagens do usuário)
- Aplicar reações sem validar primeiro se o message_id está correto
- Suppor que um número em \"mensagem X\" se refere a um ID histórico em vez de tratar a mensagem contendo a designação como o alvo

O protocolo estabelecido exige:
1. Quando solicitado a reagir a \"essa mensagem X\", tratar a mensagem contendo essa designação como o alvo exato
2. Sempre obter os IDs da sessão (CHAT_ID, MESSAGE_ID, THREAD_ID) 
3. Testar primeiro com uma reação conhecida por ser válida (👍) para validar os IDs
4. Só então aplicar a reação solicitada se o teste benigno retornar sucesso
5. Nunca pular direto para a aplicação da reação sem a etapa de validação

Essas correções visam eliminar suposições perigosas e estabelecer um padrão de operação baseado em verificações técnicas concretas.
